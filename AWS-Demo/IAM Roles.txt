An IAM role is a set of permissions that can be assumed by a service (like EC2, Lambda, etc.) or a user.
Roles are used to delegate access without using long-term credentials like access keys.

For example:
An EC2 instance might need to access S3 buckets, so you assign it a role with AmazonS3ReadOnlyAccess.
